# Разработка ETL pipeline для бота помощника в ИТП

> [!tldr] ai review
> 

## Описание

Проект: Pyrus
Команда проекта:
1. группа сопровождения инфраструктуры и технической поддержки пользователей - sre-инженеры, сотрудники поддержки
2. группа автоматизации бизнес-процессов - аналитики
Описание проекта: 

Служба технической поддержки обрабатывает обращения пользователей в системе Pyrus по вопросам работы с Pyrus.
Как сейчас выстроен процесс:
Пользователь создает обращение (в контексте Pyrus обращение = задача, шаблон = форма) по определенному шаблону, где указывает с чем у него возникли трудности, описывает проблему. Созданная задача, согласно правил маршрутизации формы попадает на обработку сотрудникам поддержки. Сотрудника поддержки изучает описание проблемы. При изучении проблемы сотрудник поддержки может воспользоваться накопленной информацией на confluence, в ранее отработанных тикетах, запросить помощь у key account manager Pyrus, бизнес-аналитиков







1. служба технической поддержки пользователей сопровождает продукт Pyrus - это crm система с возможностью low-code - pyrus.com. пользователи создают тикеты в pyrus, а сотрудники поддержки консультируют и помогают только по системе pyrus.
    
2. данные хранятся в бд, к которой есть доступ
    
3. pyrus имеет api, к которому есть доступ
    

целевая система

1. база знаний хранится в rag
    
2. да, должна быть интеграция с ai. есть возможность использовать развернутую в контуре организации модель
    

требования к обработке

1. количество новых обращение не более 50 тикетов в сутки
    
2. обновлять базу знаний нужно ежедневно
    
3. категоризация нужна, так как в дальнейшем требуется посчитать полезный эффект от внедрения etl и ai помощника
